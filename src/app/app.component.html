What exactly is Angular lazy loading? Lazy loading provides only loading components of a user request when they are needed. Lazy Loading a large web application can be a great help for performance and minimizing the initial load size. The performance gains are very significant if the initialization of an object is an issue, like in case of accessing network services. This makes it ideal in use cases where network content is accessed and initialization times are to be kept at a minimum, like loading web pages (external libraries, images, large data files, etc).

    Advantage of Angular Lazy Loading
  • Reduce the initial object load of web application
  • Smaller object loads are lower resource costs for mobile applications.
  • Lazy loading provides much faster loading applications.
  • E-commerce Web applications notice lower resource costs since content is only delivered when the user needs it.

Lazy Loading may impact on SEO. One solution to by pass the issue for Search Engine crawling is to provide links to lazy loaded content. That is creating webpages out of the content accessible to search engine crawlers. That way a search engine can indexes the website and follows the links to find them.

It is not that complex to configure the Lazy Loading in latest Angular (5.2). I am using Threejs.org libraries to animate few mathematical animations. Threejs is a large library and it works nicely with Angular Lazy Loading techniques. I created feature module, feature module's routing module and then configured the routes.
